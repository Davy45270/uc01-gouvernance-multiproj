# UC01 Gouvernance multi-projets

Use case repo for UC01 Gouvernance multi-projets.

## Expression de besoin fonctionnelle
- **Objectifs** : piloter un portefeuille multi‑projets, arbitrer priorités/budgets, gérer capacités et dépendances.
- **Acteurs** : PMO, sponsors, chefs de projet, finance, direction.
- **Flux clés** : intake des initiatives → scoring (valeur/risque/effort) → arbitrage → allocation ressources → suivi jalons → comités de gouvernance → clôture.
- **Exceptions** : projets réglementaires/urgents priorisés; réallocation rapide en cas de dérive majeure.
- **Critères d’acceptation** : portefeuille consolidé à jour, visibilité charge/capacité, statut standardisé par projet, historique décisions.

## Analyse ROI
- **Gains** : réduction du temps d’arbitrage, meilleure allocation des ressources, baisse des dépassements budget/planning, reporting consolidé automatisé.
- **Coûts** : paramétrage référentiels, intégrations données (ERP/temps), conduite du changement.
- **KPI cibles** : % projets on‑time/on‑budget, délai moyen de décision, utilisation capacité, valeur portefeuille livrée, nombre d’escalades critiques.
